---
title: "Payout Request"
openapi: "POST /api/v1/payout/{merchant_code}"
---

<Warning>
  Combine all parameters into one string, separated by `&` character and then
  encrypt using [encrypt_decrypt algorithms](/api-reference/encrypt).
</Warning>

## Parameters

<ResponseField name="merchant_code" type="string" required>
  Provide by Provider
</ResponseField>
<ResponseField name="transaction_code" type="string" required>
  Generated by the operator.
  
  Must be unique for each transaction
</ResponseField>
<ResponseField name="transaction_timestamp" type="integer" required>
  Generate by the operator. 
  
  This parameter describes the transaction request
  TimeRanges. The more detailed information regardings timestamps, please visit https://www.epochconverter.com/.
  
Please note that we only process the timestamp on these limit

`min`: 1 hour before now

`max`: 5 minutes after now

</ResponseField>
<ResponseField name="payout_code" type="string" required>
  Example `P001`. Please contact the administrator to get your payment code
</ResponseField>
<ResponseField name="transaction_amount" type="double" required>
  The amount of of the transaction
</ResponseField>
<ResponseField name="user_id" type="string" required>
user ID
</ResponseField>
<ResponseField name="currency_code" type="string" required>
  Please refer to [currency list](/resources/currency)
</ResponseField>
<ResponseField name="address" type="string" required>
<Warning>required for crypto payout</Warning>
  THis is the user's crypto wallet address
</ResponseField>
<ResponseField name="back_account_number" type="string" required>
<Warning>required for (INR, CNY, VND, THB, KRW, BDT, JPY,BRL Payout)</Warning>
<Warning>for BRL used ID Card with 11 digits</Warning>
</ResponseField>
